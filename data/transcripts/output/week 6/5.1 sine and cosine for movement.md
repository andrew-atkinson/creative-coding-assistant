# Using Trigonometric Functions for Cyclical Motion

Trigonometric functions like `sin()` and `cos()` are inherently cyclical. In a mathematical context, they oscillate between fixed values—typically between -1 and 1. While these functions can be used to modulate properties like color, their most powerful application in creative coding is transforming an angular input into two-dimensional Cartesian coordinates ($x$ and $y$).

## Understanding Cyclical Functions [00:00]

A cyclical function is one that repeats its values in a periodic cycle. As seen in mathematical graphs, the sine and cosine functions oscillate between -1 and 1.

While you could use these values to change colors, a more conventional and visually impactful use is to map an input angle to spatial coordinates. By using the sine function to generate a $y$ value and the cosine function to derive an $x$ value, you can translate angular progression into movement through space.

## Setting Up the Environment [01:22]

To create smooth, intuitive movement, we need to set up a few variables and configuration settings in p5.js:

1.  **A Counter/Angle Variable**: We need a variable that increments every frame to act as our "time" or "angle."
2.  **Angle Mode**: By default, p5.js uses radians. Using `angleMode(DEGREES)` makes the math more intuitive for beginners.
3.  **Coordinate Translation**: To rotate or move objects around a central point rather than the top-left corner, we use `translate(width/2, height/2)`.

```javascript
let x;
let y;
let angle = 0;

function setup() {
  createCanvas(400, 400);
  angleMode(DEGREES); // Makes incrementing the angle more intuitive
}

function draw() {
  background(220);
  translate(width/2, height/2); // Shifts origin (0,0) to the center
  
  // Logic goes here...
  
  angle++; // Increment the angle to create continuous movement
}
```

## Scaling Amplitude [02:20]

A critical realization when using `sin()` or `cos()` is that their raw output is very small. Because the functions only return values between -1 and 1, using them directly to move an object on a 400x400 canvas will result in almost imperceptible movement.

To solve this, we use **scaling via multiplication**. By multiplying the result of the function by a larger number (the amplitude), we expand the range of motion.

```javascript
// Without scaling: movement is only 2 pixels (from -1 to 1)
y = sin(angle); 

// With scaling: movement is 200 pixels (from -100 to 100)
y = sin(angle) * 100; 
```

This is a common pattern in generative art: generating a small, controlled range of values (like `noise()` which returns 0 to 1) and then scaling them up to a usable visual range.

## Implementing Circular Motion [04:30]

When you combine sine and cosine, you can achieve perfect circular motion. While `sin()` handles the vertical oscillation ($y$), `cos()` handles the horizontal oscillation ($x$).

By applying both to a single point, the object moves in a circle because as one value reaches its peak, the other is at zero, and vice versa.

```javascript
let x;
let y;
let angle = 0;

function setup() {
  createCanvas(400, 400);
  angleMode(DEGREES);
}

function draw() {
  background(220);
  translate(width/2, height/2);

  // Calculate coordinates based on the current angle
  x = cos(angle) * 100;
  y = sin(angle) * 100;

  // Draw the moving circle
  circle(x, y, 20);

  angle++;
}
```

### Visualizing the Geometry [06:38]

To understand why this works, we can visualize the relationship between the angle and the coordinates. By drawing lines from the origin $(0,0)$ to our calculated $x$ and $y$, we can see the geometric relationship:

*   A horizontal line from $(0,0)$ to $(x, 0)$.
*   A vertical line from $(x, 0)$ to $(x, y)$.
*   The **hypotenuse** connecting $(0,0)$ directly to $(x, y)$.

This hypotenuse represents the radius of our circle. By drawing these lines, we see how the sine and cosine functions are essentially "splitting" the angular movement into two perpendicular dimensions.

```javascript
// Visualizing the components of the circle
line(0, 0, x, 0); // Horizontal component
line(x, 0, x, y); // Vertical component
line(0, 0, x, y); // The hypotenuse (radius)
```
