# Introduction to Data Formats

In creative coding, we often move from generating data algorithmically (creating our own arrays and objects) to consuming external datasets. This transition allows us to use real-world information—such as geographic coordinates or social media statistics—to drive our visualizations.

To work with external data, you must understand how that data is structured and stored in text files. There are two primary paradigms for representing data on the web: **Tabular Data** and **Hierarchical (Object-Oriented) Data**.

## Tabular Data: CSV and TSV [01:33]

Tabular data is organized into a two-dimensional structure of rows and columns, much like an Excel spreadsheet or a Google Sheet. This format is conceptually identical to a two-dimensional array: a series of lists where each element is separated by a specific character.

### Common Formats
*   **CSV (Comma-Separated Values):** Uses a comma (`,`) as the delimiter between values.
*   **TSV (Tab-Separated Values):** Uses a tab character to separate values.

### Visualizing the Structure
While we often view data in a clean, formatted table with orange headers and organized rows, the underlying text file is much simpler. 

If you were to open a `.csv` file in a basic text editor, the data for a city list would look like this:

```text
ID,Name,Lat,Long,Pop.
1,London,-50,0,8000000
2,New York,-70,-40,8000000
3,Amsterdam,-50,5,2000000
```

In a `.tsv` file, the structure remains the same, but the commas are replaced by tabs:

```text
ID	Name	Lat	Long	Pop.
1	London	-50	0	8000000
2	New York	-70	-40	8000000
3	Amsterdam	-50	5	2000000
```

### Accessing Tabular Data in Code
Because of this structure, you can access data points using two methods:
1.  **By Index:** Referencing the specific column number and row number (e.g., `data[row][column]`).
2.  **By Header:** Using the labels provided in the header row to identify specific columns.

## Hierarchical Data: JSON [04:26]

The second, and perhaps more common format for web-based projects, is **JSON (JavaScript Object Notation)**. JSON is a subset of JavaScript object syntax and is the standard for communicating data over the web.

When you interact with social media, the platform doesn't send a full graphic of a post; instead, it sends a structured object containing the text, user info, and links.

### JSON Structure
JSON is highly flexible and allows for nested objects and arrays. Unlike tabular data, which is strictly two-dimensional, JSON can represent complex, multi-layered relationships.

A dataset of cities represented in JSON looks like this:

```json
[
  {
    "ID": 0,
    "Name": "London",
    "Long": 0,
    "Lat": -50
  },
  {
    "ID": 1,
    "Name": "New York",
    "Long": -70,
    "Lat": -40
  }
]
```

### Key Rules of JSON Syntax
To ensure a JSON file is valid, you must follow strict rules:
*   **Keys:** All keys (e.g., `"Name"`) **must** be enclosed in double quotes.
*   **Separators:** A colon (`:`) must separate the key from its value.
*   **Data Types:** Values can be numbers (which do not require quotes) or strings (which do).
*   **Nesting:** JSON can contain nested objects and arrays, making it very powerful for complex data.
*   **Limitations:** JSON is strictly for data; you cannot include executable functions within a JSON file. If you need to store code-like logic, it must be stored as a string.

### Navigating JSON
Because JSON mirrors JavaScript objects and arrays, you navigate it using the same methods you already know: **indices** for arrays or **key names** for objects.

```javascript
// Example of accessing data in a JSON array
let cityName = cities[0].Name; // Returns "London"
```

## Summary of Data Formats [06:35]

| Feature | CSV / TSV | JSON |
| :--- | :--- | :--- |
| **Structure** | Tabular (Rows/Columns) | Hierarchical (Nested Objects) |
| **Analogy** | Spreadsheet / 2D Array | JavaScript Object / Array |
| **Delimiter** | Comma (`,`) or Tab (`\t`) | Colon (`:`) and Commas (`,`) |
| **Complexity** | Simple, flat structures | Complex, nested structures |
