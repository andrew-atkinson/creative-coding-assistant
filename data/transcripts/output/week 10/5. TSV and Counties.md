# Data and Visualization: From JSON to Tabular Datasets

This resource explores the practicalities of working with real-world datasets in a creative coding environment. We move from simple, structured JSON objects to more complex tabular data (TSV/CSV), focusing on the technical challenges of parsing, data integrity, and mapping geographic coordinates to a 2D canvas.

## Understanding Data Formats [00:27]

When creating data visualizations, you will primarily encounter two types of structured data formats. Understanding the difference is crucial for choosing the correct loading function in p5.js.

### JSON (JavaScript Object Notation)
JSON is a structured format where data is organized into nested objects and arrays. Each "record" in a JSON file is an object with specific keys (like `latitude` or `population`) and corresponding values.

**Example JSON structure:**
```json
{
  "city": "New York",
  "latitude": 40.7127,
  "longitude": -74.0059,
  "population": "8405837"
}
```

### TSV and CSV (Tabular Data)
CSV (Comma Separated Values) and TSV (Tab-Separated Values) are tabular formats, similar to a spreadsheet in Excel or Google Sheets. Data is organized into rows and columns.

*   **Rows:** Represent individual records (e.g., one row per county).
*   **Columns:** Represent specific properties of those records (e.g., Name, Population, Latitude).
*   **Delimiters:** A character (a comma or a tab) separates the values within a row.

## Working with Geographic Data [01:30]

When visualizing geographic data, you must first determine the boundaries of your dataset to map coordinates correctly. This is done by iterating through the data and finding the minimum and maximum values for latitude and longitude.

```javascript
let cities;
let minLat = 90;
let maxLat = -90;
let minLong = 180;
let maxLong = -180;

function preload() {
  loadJSON("assets/cities.json", onLoad, onError);
}

function setup() {
  createCanvas(600, 400);
  
  // Calculate bounds to prepare for mapping coordinates
  for (let city of cities) {
    if (city.latitude < minLat) {
      minLat = city.latitude;
    }
    if (city.latitude > maxLat) {
      maxLat = city.latitude;
    }
    if (city.longitude < minLong) {
      minLong = city.longitude;
    }
    if (city.longitude > maxLong) {
      maxLong = city.longitude;
    }
  }
}
```

The canvas shows a scatter plot of red dots. These dots represent the cities, and their positions are determined by mapping these `min`/`max` values to the width and height of the canvas.

## Navigating Real-World Datasets [03:06]

Real-world data, such as that provided by the U.S. Census Bureau, is often "messy." When working with government datasets (like Gazetteer files), keep the following in mind:

1.  **Data Volatility:** Data is a model of the world, and the world changes. Boundaries for congressional districts or county lines shift over time.
2.  **File Formats:** Large datasets are often provided as compressed `.zip` files containing `.txt` files.
3.  **The "Tab" Character:** In TSV files, the separation between columns is a single tab character. While it looks like empty space in a spreadsheet, it is a specific character that the computer uses to parse the data.

### Inspecting Tabular Data
When opening a `.txt` or `.csv` file in a spreadsheet program, you might see columns like `GEOID`, `NAME`, and various numerical values. 

**Technical Note on Data Integrity:**
In a spreadsheet, it may appear that columns are perfectly aligned. However, if a data field (like a county name) is longer than expected or contains unexpected characters, it can "knock the data out of sequence," causing subsequent columns to shift. This is why understanding the underlying delimiter (the tab or comma) is more important than how it looks in a spreadsheet viewer.

## Implementation Workflow [06:01]

To transition from working with a simple JSON file to a more complex tabular dataset, follow this workflow:

1.  **Duplicate your sketch:** Always work on a copy when experimenting with new data types to avoid breaking your existing logic.
2.  **Prepare the assets:** Download your dataset, unzip it if necessary, and move the file into your project's `assets` folder.
3.  **Rename for clarity:** Use simple, descriptive filenames (e.g., `counties.txt`) to avoid confusion during the loading process.
4.  **Load the data:** Use `loadTable()` for TSV/CSV files, ensuring you specify the `"header"` parameter if the file contains a header row.

```javascript
// Example of loading tabular data
function preload() {
  loadTable('assets/counties.txt', 'text', 'header');
}
```

# Working with Tabular Data in p5.js

This lesson explores how to transition from simple JSON objects to more complex tabular datasets (like `.csv`, `.tsv`, or `.txt` files) using the `p5.js` library. We will cover data loading, parsing tabular structures into nested arrays, and navigating multi-dimensional data.

## Loading Tabular Data [06:58]

While `loadJSON()` is ideal for structured, nested objects, tabular data requires a different approach. To handle files organized in rows and columns, we use the `loadTable()` function.

### The `loadTable()` Syntax
The `loadTable()` function requires specific parameters to correctly interpret the file:

```javascript
counties = loadTable(filename, extension, [header], [callback], [errorCallback]);
```

*   **`filename`**: The path to the file (e.g., `"assets/counties.txt"`).
*   **`extension`**: The data format. Common values include `"csv"` (comma-separated), `"tsv"` (tab-separated), or `"ascii"`.
*   **`header`**: A boolean (or string indicator) that tells the program if the first row contains column names. If true, p5.js will skip that row when iterating through data so you don't accidentally try to perform math on a column title.
*   **`callback`**: A function to run once the data has successfully loaded.

### Handling File Extensions
When working with raw text files (`.txt`), you must explicitly tell p5.js how the data is delimited. For example, if your text file uses tabs to separate values, you must pass `"tsv"` as the extension.

## Debugging Data Loading Errors [09:03]

When working with large datasets, you may encounter errors such as `Uncaught RangeError: Maximum call stack size exceeded`. This often happens when the browser's memory is overwhelmed by attempting to log or process a massive, complex object all at once.

If you encounter issues where data isn't appearing in the console, use the **Chrome Developer Tools** (accessible via a keyboard shortcut or through the "Inspect" menu) to access the **JavaScript Console**. This provides a clearer view of exactly what is being loaded and where the logic is failing.

## Transforming Tables into Arrays [10:00]

By default, `loadTable()` returns a `p5.Table` object. While this is useful for accessing specific cells via row and column indices, it can be cumbersome for complex generative art logic. 

To make the data easier to manipulate with standard JavaScript loops, you can convert the table into a standard **2D Array** using the `.getArray()` method.

```javascript
let countiesArray;

function setup() {
  createCanvas(600, 400);
  // Convert the p5.Table object into a nested JavaScript array
  countiesArray = counties.getArray();
}
```

### Understanding the 2D Array Structure
Once converted, your data becomes a "nested array"—an array where every element is itself another array. This structure represents the rows and columns of your original file:

*   **The Outer Array**: Represents the entire dataset (the collection of all rows).
*   **The Inner Array**: Represents a single row within the dataset.

#### Visualizing Data Indexing
If you have a table representing cities, the structure looks like this:

| Index | [0] (ID) | [1] (Name) | [2] (Lat) | [3] (Long) |
| :--- | :--- | :--- | :--- | :--- |
| **[0]** | 1 | London | -50 | 0 |
| **[1]** | 2 | New York | -70 | -40 |

To access a specific piece of data, you use double indexing: `array[row][column]`.

*   `countiesArray[0]` returns the **entire first row**: `[1, "London", -50, 0]`
*   `countiesArray[0][1]` returns the **second element of the first row**: `"London"`
*   `countiesArray[1][2]` returns the **third element of the second row**: `-70`

### Implementation Workflow
When working with large datasets (e.g., 3,000+ rows), the workflow follows this pattern:
1.  **Load** the file using `loadTable()`.
2.  **Convert** the table to a 2D array using `.getArray()`.
3.  **Iterate** through the outer array to access each row.
4.  **Access** specific indices within that row (like Latitude and Longitude) to map them to your canvas coordinates.

# Mapping Geographic Data with p5.js

This guide demonstrates how to transform raw tabular data into a visual geographic representation using the `p5.js` library, specifically focusing on layering multiple datasets to create a sense of depth and structure.

## Transforming Tabular Data into Arrays [13:48]

When working with data loaded via `loadTable()`, p5.js returns a `p5.Table` object. While useful for specific table methods, it is often easier to manipulate data when it is converted into a standard JavaScript array.

By using the `.getArray()` method, you convert the table object into an "array of arrays." This makes it much simpler to iterate through rows and access specific columns by their index.

```javascript
let cities, counties, countiesArr;

function preload() {
  loadJSON('assets/cities.json', onLoad, onError);
  // Loading a TSV file with headers enabled
  counties = loadTable("assets/counties.txt", "tsv", "header");
}

function setup() {
  createCanvas(600, 400);
  // Convert the table object into a standard array for easier manipulation
  countiesArr = counties.getArray();
}
```

## Layering Data for Visual Depth [14:04]

To create a visualization with depth, you should draw your background or "base" layer first and then overlay more detailed data on top. In this demonstration, we use the `countiesArr` to create a structural outline and then overlay individual cities.

### Mapping Coordinates
To turn geographic coordinates (latitude and longitude) into screen pixels ($x$ and $y$), we use the `map()` function. 

**Note on Padding:** When mapping geographic data, it is often helpful to add a small buffer (e.g., `+ 5` or `- 5`) to the data values to ensure points near the extreme edges of the dataset aren't cut off by the canvas boundaries.

```javascript
function draw() {
  background(0); // Black background

  // Layer 1: The "Base" - Drawing Counties
  fill(0, 0, 100); // A dark blue for the base layer
  noStroke();

  for (let countyEl of countiesArr) {
    // Accessing data by index: 
    // In this dataset, index 9 is Longitude (X) and index 8 is Latitude (Y)
    let x = map(countyEl[9], -123.2620435 - 5, -70.2553529 + 5, 0, width);
    let y = map(countyEl[8], 48.74908 + 5, 25.4687224 - 5, 0, height);
    ellipse(x, y, 3);
  }

  // Layer 2: The "Detail" - Drawing Cities
  fill(255, 0, 0); // Red for the city points
  for (let city of cities) {
    // Using object properties (city.longitude) instead of indices
    let x = map(city.longitude, -123.2620435 + 5, -70.2553529 + 5, 0, width);
    let y = map(city.latitude, 48.74908 + 5, 25.4687224 - 5, 0, height);
    ellipse(x, y, 2);
  }
}
```

The resulting visual shows a dark blue "cloud" of points representing the density and shape of counties, with bright red dots representing specific cities layered on top. This provides a much more structured and recognizable outline of the United States than city data alone.

# Data Normalization and Type Coercion in Data Visualization

When working with real-world datasets, the transition from raw data to visual representation often involves navigating "messy" technical hurdles. This guide explores the process of finding data extrema (minimums and maximums) to prepare for mapping, while addressing a critical pitfall: JavaScript's type coercion.

## Finding Data Extrema [20:09]

To map data values (like population) to visual properties (like dot size or color), you first need to determine the range of your dataset. This requires finding the minimum and maximum values within a collection of objects.

To do this, initialize your boundary variables with extreme starting values:
*   **Minimum:** Start with a very large number (or the first value in your set).
*   **Maximum:** Start with zero.

```javascript
let minPop = 1000000; // A large starting value
let maxPop = 0;       // The lowest possible population

for (let city of cities) {
  if (city.population < minPop) {
    minPop = city.population;
  }
  if (city.population > maxPop) {
    maxPop = city.population;
  }
}

console.log("population:", minPop, maxPop);
```

## The Pitfall: Type Coercion and String Comparison [22:03]

A common issue when loading data from external files (like JSON or CSV) is that numerical values are often parsed as **strings** rather than **numbers**. This leads to unexpected results due to **type coercion**.

### The Problem: Lexicographical vs. Numerical Sorting
If you attempt to find the maximum value of a dataset where numbers are stored as strings, JavaScript performs a lexicographical (alphabetical) comparison rather than a numerical one.

For example, if your console shows:
`population: 100050 99889`

This is incorrect. In a numerical comparison, $100,050$ is much smaller than $99,889$. However, in a string comparison, the character `"9"` has a higher Unicode/ASCII value than the character `"1"`. Therefore, JavaScript identifies the string starting with `9` as "larger," regardless of the actual magnitude of the number.

### The Solution: Explicit Type Conversion
To ensure mathematical accuracy, you must explicitly convert the string values into numbers using the `Number()` constructor before performing comparisons.

```javascript
// The incorrect way (comparing strings)
if (city.population > maxPop) { 
  maxPop = city.population; // Reassigning a string to maxPop
}

// The correct way (converting to numbers)
if (Number(city.population) > maxPop) {
  maxPop = Number(city.population);
}
```

## Optimizing Data Processing [25:00]

When performing type conversion inside a loop, calling `Number()` multiple times on the same variable is inefficient and makes the code harder to maintain. A cleaner approach is to assign the converted value to a local variable at the start of each iteration.

```javascript
let minPop = 1000000;
let maxPop = 0;

for (let city of cities) {
  // Convert once per iteration for efficiency and readability
  let pop = Number(city.population); 

  if (pop < minPop) {
    minPop = pop;
  }
  if (pop > maxPop) {
    maxPop = pop;
  }
}

console.log("population:", minPop, maxPop);
```

## Managing Variable Scope [25:34]

When calculating these values, the location of your code determines its visibility to other functions. 

If you calculate `minPop` and `maxPop` inside the `setup()` function, they are **locally scoped** to that function. This means you cannot access them inside the `draw()` function or other parts of your program to perform mapping.

To make these values available throughout the entire sketch, you must declare them as **global variables** at the top of your script:

```javascript
// Global Scope: Accessible everywhere
let minPop;
let maxPop;

function setup() {
  // ... data loading and calculation logic goes here ...
}

function draw() {
  // Now minPop and maxPop are visible here for mapping!
}
```

# Data Visualization: Mapping Population to Color

This guide explores the practical challenges of transforming raw geographic and statistical data into a visual representation using `p5.js`. We focus on the transition from simple coordinate mapping to sophisticated data normalization and handling real-world data distributions.

## Calculating Data Extremes [26:00]

Before we can map data to visual properties like color or size, we must first determine the range of our dataset. This involves iterating through the data to find the minimum and maximum values for both geographic coordinates (latitude/longitude) and statistical attributes (population).

```javascript
// Finding the bounds of our dataset
for (let city of cities) {
  let pop = Number(city.population); // Ensure population is treated as a number
  if (pop < minPop) {
    minPop = pop;
  }
  if (pop > maxPop) {
    maxPop = pop;
  }
}

console.log("population:", minPop, maxPop);
```

The console output allows us to verify these bounds. For example, seeing `population: 36877 8405837` confirms the scale of our data, which is essential for the next step: mapping.

## Mapping Data to Visual Properties [26:30]

Once we have our data ranges, we use the `map()` function to translate those values into screen-space coordinates and aesthetic properties.

### Coordinate Mapping
To turn latitude and longitude into $x$ and $y$ pixels, we map the geographic values to the canvas `width` and `height`. We often add a small buffer (e.g., `+ 5` or `- 5`) to the data bounds to ensure points near the edges of the map aren't cut off by the canvas border.

```javascript
let x = map(city.longitude, minLong - 5, maxLong + 5, 0, width);
let y = map(city.latitude, maxLat + 5, minLat - 5, 0, height);
```

### Color Mapping and the "Type Coercion" Pitfall
We can also map population to a color range. A common mistake is attempting to use the raw data directly, but because many datasets (like JSON) store numbers as strings, we must explicitly convert them using `Number()`.

```javascript
// Mapping population to a red color scale (0 to 255)
let col = map(Number(city.population), minPop, maxPop, 50, 255);
fill(col, 0, 0); // Applying the mapped value to the Red channel
```

## Solving the "Long Tail" Problem [27:30]

A significant challenge in data visualization is the **non-linear distribution** of real-world data. In a dataset of cities, population follows an exponential curve: the largest city might have 8 million people, while the next few are significantly smaller (e.g., 3.8 million, then 1.5 million).

If we use a linear map from the absolute minimum to the absolute maximum, most cities will end up with very similar values.

### The Problem: Linear Mapping
When mapping from `minPop` to `maxPop`, the vast majority of cities fall into a very narrow range at the bottom end of the scale. This results in a visualization where almost every dot looks identical (e.g., very dark red), making the data appear visually "flat."

### The Solution: Flattening the Curve
To create visual variety, we can "flatten" the top of our scale. Instead of mapping up to the absolute maximum (8 million), we can cap our input range at a lower threshold, such as 1,000,000. This ensures that any city with over a million people is treated as "maximum brightness," while the variation between smaller cities becomes more visually apparent.

```javascript
// Flattening the scale: anything over 1,000,000 is treated as "max"
let col = map(Number(city.population), minPop, 1000000, 0, 255);
fill(col, col, col); // Mapping to grayscale for a more subtle effect
```

## Managing Visual Layers and Sorting [31:27]

When drawing many overlapping points, the order in which they are drawn matters. If we draw large dots first and small dots later, the small dots will appear on top. However, if our data is sorted from largest to smallest, the large "outlier" cities might visually overwhelm and hide all the smaller data points.

To ensure that small, detailed points remain visible even when they overlap with large ones, you should **sort your data array** so that the largest elements are drawn first and the smallest elements are drawn last.

```javascript
// Conceptual approach: Sort cities so largest are drawn first, 
// allowing smaller dots to be layered on top.
cities.sort((a, b) => Number(b.population) - Number(a.population));
```

# Data Visualization: Sorting and Scaling Complex Datasets

This guide explores the practical challenges of transforming raw data into meaningful visual representations. We focus on two critical aspects: ensuring correct drawing order through array sorting and managing extreme data distributions using non-linear mathematical scaling.

## Sorting Data for Visual Depth [32:09]

When visualizing data points (like cities on a map), the order in which you draw them matters. If you draw large points first and small points second, the smaller points will overlap and hide the larger ones. To ensure that large data points remain visible on top of smaller ones, you must sort your array before iterating through it.

### Implementing a Custom Sort Function
To sort an array of objects by a specific property (like population), you use the `Array.prototype.sort()` method with a custom comparison function.

The instructor demonstrates using an **arrow function** to compare the population values of two objects, `a` and `b`:

```javascript
// Sort cities so that the largest populations are at the end of the array.
// This ensures they are drawn last, appearing on top of smaller dots.
cities.sort((a, b) => {
  return a.population - b.population;
});
```

**Key Technical Insights:**
*   **In-place Sorting:** The `.sort()` method sorts the array "in place," meaning it modifies the original array rather than creating a new one. Therefore, you do not need to reassign it (e.g., `cities = cities.sort(...)` is redundant).
*   **The Comparison Logic:** The function `(a, b) => a.population - b.population` tells JavaScript how to compare two elements. If the result is negative, `a` comes before `b`; if positive, `b` comes before `a`.
*   **Visual Result:** By sorting from smallest to largest, the smaller dots are drawn first, and the larger, more significant data points are layered on top.

## Managing Data Skew with Non-Linear Scaling [34:00]

A common problem in data visualization is the "long tail" distribution. In many datasets—such as city populations—a few values are massive (e.g., 8 million), while the vast majority of values are relatively small (e.g., 30,000 to 100,000).

### The Problem with Linear Mapping
If you map population linearly to a color range (e.g., 0 to 255), the massive outliers will "squash" the rest of the data. Most cities will end up with nearly identical, dark colors because their population values are so small compared to the maximum.

```javascript
// Linear mapping: This often fails for skewed data
let col = map(Number(city.population), minPop, maxPop, 0, 255);
```

### The Solution: "Flattening the Curve" with `Math.sqrt()`
To make the visual variance more legible, you can apply a mathematical transformation to "flatten" the distribution. Using the **square root** of the value reduces the impact of extreme outliers while still maintaining a relationship between the original value and its visual representation.

The instructor demonstrates calculating a new range based on the square roots of the population values:
*   **Smallest city:** $\sqrt{36,000} \approx 190$
*   **Largest city:** $\sqrt{8,400,000} \approx 2,900$

By mapping the square root of the population to your color range, you create a much more effective visual gradient:

```javascript
for (let city of cities) {
  // Transform the population using Math.sqrt to handle the exponential distribution
  let popValue = Number(city.population);
  let col = map(Math.sqrt(popValue), 200, 2900, 100, 255);
  
  fill(col, 0, 0); // Mapping to a red color scale
  // ... drawing logic
}
```

### Visualizing the Mathematical Shift
Imagine a graph of population vs. color:
1.  **Linear Scale:** A steep, nearly vertical drop-off where almost all points are clustered at one end of the color spectrum.
2.  **Square Root Scale:** A much shallower, more gradual curve. This "stretches" the lower end of the data, allowing you to see subtle differences between smaller cities while still distinguishing the largest ones.

By using this approach, the visualization becomes much more informative, revealing a "median" range of colors rather than just a few extreme outliers.

# Data Visualization: Mapping Geographic Datasets

## Visualizing City Data with Non-Linear Scaling [38:30]

When visualizing data like population, a common challenge is the "long tail" effect: a few massive cities can dwarf all other data points, making it impossible to see the variance among smaller cities. To solve this, we can use `Math.sqrt()` to "flatten the curve," allowing smaller values to remain visually distinct.

The following code demonstrates how to map city coordinates to the canvas and use a square root transformation on population to determine color intensity.

function setup() {

function draw() {

## Implementing Categorical Color Logic [40:12]

A powerful way to add visual structure is to change the color based on a categorical variable, such as a "State" column. We can achieve this by tracking the "current state" while looping through the data and triggering a color change only when the value changes.

```javascript
let currentState = ""; // Tracks the state of the previous iteration

function draw() {
  background(0);
  
  for (let countyEl of countiesArr) {
    // Check if the current element's state is different from the last one processed
    if (currentState != countyEl[0]) {
      // If it's a new state, pick a random color for this group
      fill(random(100), random(100), random(200));
      // Update currentState so the next county in this state doesn't trigger a new color
      currentState = countyEl[0];
    }

    let x = map(countyEl[9], -123.2620435 + 5, -70.2553259 + 5, 0, width);
    let y = map(countyEl[8], 48.74908 + 5, 25.4687224 - 5, 0, height);
    ellipse(x, y, 3);
  }
}
```

**Visual Result:** This logic produces a "map" effect where clusters of dots share the same color, creating distinct visual blocks corresponding to geographic regions (e.g., a block of blue for one state, then a block of green for the next).

## Aesthetic Refinement and Debugging [42:30]

The transition from a technical visualization to "data art" involves fine-tuning the scale and style.

1.  **Color Palettes:** Instead of purely random colors, limiting the RGB ranges (e.g., `random(50), random(100), random(200)`) can create a cohesive color palette (like blues and greens) rather than a chaotic mix.
2.  **Point Size:** The size of the elements (e.g., `ellipse(x, y, 1.8)`) significantly impacts the "feel" of the map. Smaller points create a more delicate, granular texture.
3.  **Stroke vs. Fill:** Using `noStroke()` can prevent the visualization from looking cluttered, especially when points are densely packed.
4.  **Debugging with the Console:** When data doesn't appear as expected, use `console.log(yourArray)` to inspect the structure of your data and ensure you are accessing the correct indices (e.g., `countyEl[0]` for state, `countyEl[8]` for latitude).

# Data Visualization and Aesthetic Decision-Making

This session explores the transition from raw data to visual representation, focusing on how mathematical transformations and aesthetic choices impact the legibility of a data-driven visualization.

## Mapping Data to Visual Properties [45:00]

When visualizing geographic data, we must map disparate numerical values (like population) to visual properties like color and size. A common challenge is the "long tail" of data, where a few extreme outliers can make all other data points visually indistinguishable.

### Handling Exponential Data with Non-Linear Scaling
If you map population directly to dot size, a city with 10 million people will appear massive, while a city with 100,000 people becomes an invisible speck. To "flatten the curve" and make smaller data points visible, use `Math.sqrt()` to transform the scale.

```javascript
for (let city of cities) {
  // Using Math.sqrt to ensure smaller populations are still visible
  let col = map(Math.sqrt(Number(city.population)), 200, 2500, 100, 255);
  
  noStroke();
  fill(col, 0, 0); // Mapping population to a red color channel

  // Map longitude and latitude to canvas x and y coordinates
  let x = map(city.longitude, -123.2620435 + 5, -70.2553259 + 5, 0, width);
  let y = map(city.latitude, 48.74908 + 5, 25.4687224 - 5, 0, height);
  
  ellipse(x, y, col/8); // Using the calculated value to determine size
}
```

### The Importance of Type Conversion
When pulling data from external files (like JSON or TSV), numbers are often treated as strings. Performing mathematical operations on strings can lead to logical errors or failed comparisons. Always wrap your data values in `Number()` before performing calculations:

```javascript
// Correct way to ensure population is treated as a number
let pop = Number(city.population);
```

## Managing Visual Complexity and Scale [46:00]

As a visualization grows in complexity, the "scale of representation" becomes a primary concern. A common issue is that elements become lost when the user zooms out or when the canvas size changes.

### Aesthetic Refinement: Size vs. Opacity
When a visualization feels "cluttered" or "lost," you face a design choice: make the elements larger or change their opacity. 

*   **Increasing Size:** Makes individual data points more distinct but can lead to overlapping "blobs" that obscure the overall pattern.
*   **Adjusting Opacity:** Using lower opacity (e.g., `col/16` or a fixed low value) allows for overlapping points to create new colors, revealing density through color accumulation.

The instructor demonstrates an iterative process of adjusting these values to find a "resolved" look:
1.  **Initial state:** Small, bright dots that disappear when zoomed out.
2.  **Attempt 1:** Increasing dot size but reducing opacity to prevent overwhelming the screen.
3.  **Attempt 2:** Using `windowWidth` to ensure the canvas scales with the browser, maintaining proportions.

```javascript
function setup() {
  // Scaling canvas to window width while maintaining aspect ratio
  createCanvas(windowWidth, windowWidth * 0.6);
}
```

## Data Layering and Sorting [47:30]

To ensure that smaller data points aren't completely covered by larger ones, you should control the drawing order. By sorting your data array before iterating through it, you can ensure that "background" elements are drawn first and "foreground" elements (the most significant data points) are drawn last.

```javascript
// Sort cities by population in ascending order 
// This ensures smaller dots are drawn first and larger ones stay on top
cities.sort((a, b) => a.population - b.population);

for (let city of cities) {
  // Drawing logic here...
}
```

### Summary of the Workflow
1.  **Load Data:** Use `loadJSON()` or `loadTable()`.
2.  **Clean/Convert:** Ensure all numerical values are cast via `Number()`.
3.  **Sort:** Order the array to manage visual layering.
4.  **Map:** Translate geographic and statistical data into $x, y$, color, and size using `map()` and non-linear functions like `Math.sqrt()`.
5.  **Tinker:** Iteratively adjust opacity and scale to balance detail with legibility.
