# Asynchronous Data Loading with Callbacks in p5.js

In web-based creative coding, loading external data is an **asynchronous** operation. This means the program does not wait for the file to finish downloading before moving to the next line of code. To handle this, we use **callback functions**—functions that are executed only when a specific event occurs (such as a successful data load or an error).

## The Baseline: Implicit Assignment [00:00]

The simplest way to load JSON in p5.js is to assign the result of `loadJSON()` directly to a global variable within the `preload()` function.

```javascript
let cities;

function preload() {
  cities = loadJSON('assets/cities.json');
}

function setup() {
  createCanvas(400, 400);
  console.log(cities); // Logs the loaded data object
}

function draw() {
  background(220);
}
```

While this works for simple scripts, it doesn't give us explicit control over what happens the moment the data arrives or if something goes wrong during the download.

## Implementing Explicit Callbacks [01:00]

To gain more control, we can pass function names as arguments to `loadJSON()`. This allows us to define specific logic for a "success" state and an "error" state.

### The Success Pattern
When the file loads successfully, we pass a function (e.g., `onLoad`) that receives the data as a parameter (often named `res` for "response").

```javascript
function preload() {
  // We pass the function name 'onLoad' without parentheses
  loadJSON('assets/cities.json', onLoad);
}

function onLoad(res) {
  // 'res' contains the parsed JSON data
  cities = res; 
  console.log("onLoad success!:", cities[0]);
}
```

**Technical Insight: Passing vs. Calling Functions**
When passing a callback to `loadJSON`, you must pass the function name itself (`onLoad`) rather than calling it (`onLoad()`). If you include parentheses, the function executes immediately during the `preload` stage, rather than waiting for the data to arrive.

When running this code, the console will display the first object in the array (e.g., `city: "New York"`), confirming that the data structure is accessible and correctly parsed.

### The Error Pattern
To make your code robust, you should also handle cases where the file is missing or the path is incorrect. You can do this by adding a second callback function for errors.

```javascript
function preload() {
  // Syntax: loadJSON(path, successCallback, errorCallback);
  loadJSON('assets/cities.json', onLoad, onError);
}

function onError(res) {
  // 'res' here is an error object containing details about the failure
  console.log("onError:", res);
}
```

## Debugging Asynchronous Errors [02:28]

If a file path is broken (for example, if you accidentally delete a letter in the filename), `loadJSON` will trigger the error callback. 

When an error occurs, the browser console provides a diagnostic object. In p5.js, this often results in an error message such as:
`Error: [object ReadableStream]`

The console will typically suggest checking if the file path is correct or ensuring you are running the code through a local server. By using `console.log("onError", res)`, you can inspect this error object to understand exactly why the data failed to load.

## Summary of Workflow [03:15]

By using callbacks, you transition from "hoping" the data is there to explicitly managing the program's state:

1.  **`loadJSON(path, onLoad, onError)`**: Initiates the request.
2.  **`onLoad(res)`**: Executes only when the data is successfully retrieved; used to assign data to global variables.
3.  **`onError(res)`**: Executes only if the request fails; used to debug and prevent the program from crashing silently.
