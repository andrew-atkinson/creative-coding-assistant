# Data Ingestion: From JSON to p5.js

This resource covers the process of finding, inspecting, and loading structured data into a p5.js environment for use in generative art and data visualization.

## Understanding JSON Structure [00:11]

Before writing code, it is essential to understand the structure of your data source. A common format for web-based data is **JSON (JavaScript Object Notation)**. 

When inspecting a JSON file—such as the `cities.json` example used in this demonstration—you are looking for an array of objects. Each object represents a single data entry (e.g., one city) and is separated by commas.

### Key Characteristics of JSON:
*   **Key-Value Pairs:** Data is stored in pairs. The "key" (the property name) and the "value" are separated by a colon `:`.
*   **Quotes:** In JSON, all keys must be wrapped in double quotes (e.g., `"city"`).
*   **Data Types:** 
    *   **Strings:** Text values must be in quotes (e.g., `"New York"`).
    *   **Numbers:** Numeric values like latitude and longitude should be stored as raw numbers (e.g., `40.7127837`) to allow for mathematical operations without further parsing.
    *   **Inconsistencies:** Be aware that real-world data is often "dirty." For example, a population might be stored as a string (`"84058537"`) rather than a number, or it might contain non-numeric characters like percent signs (`"4.8%"`).

The following is a representation of the `cities.json` structure:

```json
[
  {
    "city": "New York",
    "growth_from_2008_to_2013": "4.8%",
    "latitude": 40.7127837,
    "longitude": -74.0059413,
    "population": "84058537",
    "rank": "1",
    "state": "New York"
  },
  {
    "city": "Los Angeles",
    "growth_from_2008_to_2013": "4.8%",
    "latitude": 34.0522342,
    "longitude": -118.2436049,
    "population": "38043307",
    "rank": "2",
    "state": "California"
  }
]
```

## Data Sourcing and Cleaning [01:37]

Data is often sourced from web scraping or public repositories like GitHub. When working with scraped data, you may encounter several challenges:

1.  **Unorganized Data:** Datasets are not always sorted in a meaningful way (e.g., by population or rank).
2.  **Residual Artifacts:** Scraping data from sites like Wikipedia can leave behind "noise," such as footnote references or extra characters within a string.
3.  **Complexity:** Data might include additional layers, such as time zone information (e.g., `"America/Chicago"`), which adds complexity to your parsing logic.

**Pro-tip:** Use a browser extension like **JSON Formatter**. This allows you to view raw JSON in a "pretty" format, making it easier to collapse and expand nested structures for visual inspection.

## Managing Data in p5.js [04:31]

Once you have your data file, the most efficient workflow is to download it and upload it directly into your coding environment's file sidebar.

### Organizing the Workspace
To keep projects tidy, create an `assets` folder within your project directory and upload your `.json` files there. 

*Note: The p5.js Web Editor has a file upload limit (approximately 5MB), so be mindful of the size of your datasets.*

### The Importance of `preload()`
When loading external files, you should use the `preload()` function rather than `setup()`. 

**The Insight:** JavaScript is non-deterministic. If you attempt to access data in `setup()` before the file has finished downloading, your code will crash because the variable is still undefined. The `preload()` function guarantees that the data loading process is attempted and completed before the `setup()` and `draw()` loops begin.

### Implementing `loadJSON()`
To bring the data into your sketch, use the `loadJSON()` function. This function accepts three arguments:
1.  **Path:** The file path to the asset (e.g., `'assets/cities.json'`).
2.  **Success Callback:** A function that runs if the data loads successfully.
3.  **Error Callback:** A function that runs if there is a network or syntax error.

The implementation looks like this:

```javascript
let cities; // Variable to hold the loaded data

function preload() {
  // loadJSON(path, successCallback, errorCallback)
  cities = loadJSON('assets/cities.json'); 
}

function setup() {
  createCanvas(400, 400);
  // Once preload is done, 'cities' is ready to be used here
  console.log(cities); 
}

function draw() {
  background(220);
}
```

### Debugging Data Loading
If your sketch runs but nothing appears on the canvas, it does not necessarily mean the data failed to load. It may simply mean you haven't written the code to *draw* the data yet. 

Always use `console.log(yourVariableName)` inside `setup()` to verify that the data object is present in the browser's memory. If the console shows the expected array of objects, your ingestion was successful, and you can proceed to parsing and visualization.

# Data Ingestion and Asynchronous Loading in p5.js

This resource explores how to bring external data into a creative coding environment using the JSON format and managing the complexities of asynchronous web requests.

## Understanding `loadJSON()` [06:34]

To create data-driven visualizations, you must first ingest structured data. The `loadJSON()` function in p5.js is used to load a JSON (JavaScript Object Notation) file and convert it into a usable JavaScript object.

### Data Types and Parameters
The `loadJSON()` function can handle various data types, including strings, numbers, booleans, and null values. The function accepts two primary parameters:

1.  **`path` (String):** The location of the file. This can be a local path within your project folder, such as `'assets/data.json'`.
2.  **`successCallback` (Function, Optional):** A function that executes automatically once the data has been successfully loaded.
3.  **`errorCallback` (Function, Optional):** A function that executes if the loading process fails.

### Local vs. Remote Data
When working with data, you must be aware of **CORS (Cross-Origin Resource Sharing)**. If you attempt to load a JSON file directly from another website's URL, the browser may block the request for security reasons. To avoid these issues, it is best practice to upload the data file into your local project directory (e.g., an `assets` folder) and load it from there.

## Managing Asynchronous Behavior [07:18]

A critical concept in web-based programming is **asynchronicity**. When a computer requests data from a URL, it doesn't know how long that request will take. The network might be slow, the URL might be typed incorrectly, or the server itself might return an error.

Because computers are literal and cannot simply "wait" indefinitely for a response without freezing the entire program, we use **callbacks**.

### The Role of `preload()`
In p5.js, the `preload()` function is essential for data loading. While you can technically load data anywhere, using `preload()` guarantees that the data is fully loaded and resolved before the `setup()` and `draw()` functions begin. This prevents errors where your code tries to access properties of an object that hasn't arrived yet.

### Using Callbacks for Data Processing [09:15]
Sometimes, you don't just want to load the data; you want to process it immediately upon arrival. You can pass a custom function as the second argument to `loadJSON()`. This is particularly useful for complex datasets like GeoJSON, where you might need to iterate through nested properties to find a specific value.

**Example: Filtering GeoJSON for a Specific Value**
In this example, we load earthquake data and use a callback function (`handleData`) to find the single largest earthquake in the dataset.

```javascript
let bigQuake;

// Load the GeoJSON and preprocess it via a callback
function preload() {
  loadJSON(
    'https://earthquake.usgs.gov/earthquakes/feed/v1.0/summary/all_day.geojson', 
    handleData
  );
}

function setup() {
  createCanvas(100, 100);
  background(200);

  // Draw a circle whose size is driven by the earthquake's magnitude
  circle(50, 50, bigQuake.mag * 10);

  textAlign(LEFT, CENTER);
  textFont('Courier New');
  textSize(11);
}

// The success callback: iterates through features to find the max magnitude
function handleData(data) {
  let maxMag = 0;
  
  // We iterate over the 'features' array within the JSON object
  for (let quake of data.features) {
    // If this quake is larger than the current max, store its properties
    if (quake.properties.mag > maxMag) {
      bigQuake = quake.properties;
    }
  }
}

function draw() {
  // Display the location text extracted from the data
  text(bigQuake.place, 5, 80, 100);
}
```

## Debugging Data [10:36]

When working with external files, it is common to encounter a "silent failure"—the sketch runs and the canvas appears, but nothing happens because the data isn't being accessed correctly.

If your visual output is blank, use `console.log()` to verify that the data exists in memory.

```javascript
let cities;

function preload() {
  // Loading from a local assets folder
  cities = loadJSON('assets/cities.json');
}

function setup() {
  createCanvas(400, 400);
  // Log the object to the browser console to inspect its structure
  console.log(cities);
}

function draw() {
  background(220);
}
```

By inspecting the browser console, you can see the actual structure of your object. In the example above, the console reveals an array of objects containing keys like `growth`, `latitude`, and city names. This allows you to confirm that your file path is correct and that the data structure matches what your code expects.
