# Data Wrangling and Coordinate Mapping

This guide demonstrates the process of transforming raw geographic data into a visual representation. We will walk through loading JSON data, calculating the spatial boundaries of a dataset (bounding boxes), and mapping spherical coordinates to a 2D digital canvas.

## The Challenge of Raw Data [00:00]

When working with data, it is rarely ready for immediate visualization. Data often arrives in formats that are not "useful" for a specific design intent and may contain values that don't align with the coordinate system of your digital canvas.

The first step is **data wrangling**: the process of cleaning and transforming raw data into a structured form that can be used for visual representation.

### Asynchronous Data Loading
In p5.js, data loading is asynchronous. We use the `preload()` function to ensure that the data is fully fetched before the `setup()` and `draw()` functions execute.

```javascript
let cities;

function preload() {
  // loadJSON takes the file path, a success callback, and an error callback
  loadJSON('assets/cities.json', onLoad, onError);
}

function setup() {
  createCanvas(400, 400);
  console.log(cities);
}

function draw() {
  background(220);
}

// Callback function executed upon successful load
function onLoad(res) {
  cities = res;
  console.log("onLoad success!", cities[0]);
}

// Callback function executed if loading fails
function onError(res) {
  console.log("onError:", res);
}
```

## Mapping Spherical Coordinates to a 2D Plane [01:30]

Geographic data (Latitude and Longitude) exists on a sphere, typically represented in degrees from -180 to +180 (longitude) and -90 to +90 (latitude). A digital canvas, however, uses a Cartesian coordinate system where $x$ and $y$ are measured in pixels.

### The Bounding Box Problem
If we simply plot every coordinate from a global dataset onto a small canvas, the visualization will be problematic:
1. **Scale Issues:** A single country (like the US) might only occupy a tiny fraction of the canvas if the scale is set to the entire globe.
2. **Outliers:** Geographic outliers like Hawaii, Alaska, or Guam might fall outside the standard "rectangular" view of a country, causing data points to be clipped or lost.

To solve this, we must calculate a **bounding box**—the minimum and maximum latitude and longitude present in our specific dataset. This allows us to "zoom in" on the relevant data.

## Calculating Data Boundaries [03:33]

To find the boundaries of our dataset, we iterate through every object in our array and compare its values against running minimum and maximum variables.

### Modeling the Logic: The "Inversion" Insight
When initializing variables for a search, you must choose values that ensure the first comparison always triggers an update. 

If we are looking for a **minimum** value, we should initialize our variable to the highest possible value. If we are looking for a **maximum** value, we should initialize it to the lowest possible value.

> **Note:** In a live demonstration, the instructor intentionally inverts these values (setting `minLat` to 90 and `maxLat` to -90) to demonstrate how the logic must be structured so that the first data point encountered immediately updates both boundaries.

```javascript
let cities;
// Initializing with inverted values to ensure the first comparison updates them
let minLat = 90, maxLat = -90, minLong = 180, maxLong = -180;

function preload() {
  loadJSON('assets/cities.json', onLoad, onError);
}

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
  
  // Iterate through the cities to find the extremes
  for (let city of cities) {
    // Update Latitude boundaries
    if (city.latitude < minLat) {
      minLat = city.latitude;
    }
    if (city.latitude > maxLat) {
      maxLat = city.latitude;
    }

    // Update Longitude boundaries
    if (city.longitude < minLong) {
      minLong = city.longitude;
    }
    if (city.longitude > maxLong) {
      maxLong = city.longitude;
    }
  }
}
```

### The Iterative Comparison Logic
The logic follows this pattern for every item in the array:
1. **Check Minimum:** Is the current `city.latitude` smaller than our current `minLat`? If yes, update `minLat`.
2. **Check Maximum:** Is the current `city.latitude` larger than our current `maxLat`? If yes, update `maxLat`.

Once this loop completes, we have the exact mathematical boundaries of our data, which can then be used to map coordinates to pixels.

# Mapping Geographic Data to a 2D Canvas

This guide demonstrates the process of transforming raw geographic data (latitude and longitude) into visual points on a digital canvas. We will walk through loading JSON data, calculating the spatial boundaries of that data (the "bounding box"), and mapping those coordinates to screen pixels.

## Loading Asynchronous Data [07:00]

When working with external files like JSON, we must use asynchronous loading to ensure the data is fully downloaded before our program attempts to process it. In p5.js, this is handled within the `preload()` function using `loadJSON()`.

```javascript
let cities;

function preload() {
  // loadJSON takes the file path, a success callback, and an error callback
  loadJSON('assets/cities.json', onLoad, onError);
}

function setup() {
  createCanvas(400, 400);
  console.log(cities); // Verify data is loaded
}

function onLoad(res) {
  cities = res;
  console.log("onLoad success!", cities[0]);
}

function onError(err) {
  console.error("Error loading JSON:", err);
}
```

The `onLoad` callback function is essential; it assigns the returned JSON object to our global `cities` variable, making it available for use in `setup()` and `draw()`.

## Calculating the Bounding Box [07:13]

To map geographic coordinates to a screen, we first need to know the "extremes" of our dataset. This is known as finding the **bounding box**. We need to find the minimum and maximum values for both latitude and longitude.

### Iterative Boundary Search [07:30]

We initialize our boundary variables to the widest possible geographic ranges (Latitude: 90 to -90; Longitude: 180 to -180). We then iterate through the dataset, updating these variables whenever we find a value that exceeds our current limits.

```javascript
let minLat = 90, maxLat = -90;
let minLong = 180, maxLong = -180;

// Inside the loop processing cities:
for (let city of cities) {
  // Find minimum latitude (most southern point)
  if (city.latitude < minLat) {
    minLat = city.latitude;
  }
  // Find maximum latitude (most northerly point)
  if (city.latitude > maxLat) {
    maxLat = city.latitude;
  }
  // Find minimum longitude (most westerly point)
  if (city.longitude < minLong) {
    minLong = city.longitude;
  }
  // Find maximum longitude (most easterly point)
  if (city.longitude > maxLong) {
    maxLong = city.longitude;
  }
}

console.log(minLat, maxLat, minLong, maxLong);
```

### Validating with Real-World Context [08:30]

It is vital to verify that our calculated boundaries make sense. If we are mapping US cities, the most southern city should be something like Honolulu, Hawaii (approx. 21° N), and the most northerly should be something like Anchorage, Alaska (approx. 61° N).

By checking these values against Google Maps, we can confirm our algorithm is working correctly. If the numbers match real-world geography, our bounding box is accurate.

**Note on Data Outliers:** Real-world data often contains "outliers" that can skew your visualization. For example, if a dataset includes both mainland US cities and territories like Guam or American Samoa, the longitude range will expand significantly. Always check if your data includes unexpected locations that might compress your primary subject matter on the canvas.

## Mapping Coordinates to Screen Space [12:18]

Once we have the `min` and `max` values for our coordinates, we can use the `map()` function to translate those degrees into pixel coordinates ($x$ and $y$) on our canvas.

### The Coordinate Transformation Logic [12:30]

The `map()` function takes a value from one range and scales it to another. 

*   **For X (Longitude):** We map the longitude from its range (`minLong` to `maxLong`) to the canvas width (`0` to `width`).
*   **For Y (Latitude):** We map the latitude from its range (`minLat` to `maxLat`) to the canvas height. 

**Crucial Insight: The Y-Axis Inversion**
In geography, latitude increases as you move **up** (North). However, in computer graphics and the p5.js coordinate system, the $y$-coordinate increases as you move **down** the screen. To prevent our map from appearing upside down, we must invert the mapping:

```javascript
// Correct mapping for screen space
let x = map(city.longitude, minLong, maxLong, 0, width);
let y = map(city.latitude, maxLat, minLat, 0, height); 
// Note: maxLat is the first value in the source range to invert the axis
```

### Rendering the Visualization [13:00]

With the boundaries calculated and the mapping logic established, we can finally render the data. We set a dark background for contrast and loop through the cities to draw them as points.

```javascript
function setup() {
  createCanvas(400, 400);
  background(0); // Black background for high contrast
  fill(255, 0, 0); // Red color for the data points
  noStroke();

  for (let city of cities) {
    // 1. Calculate the mapped coordinates
    let x = map(city.longitude, minLong, maxLong, 0, width);
    let y = map(city.latitude, maxLat, minLat, 0, height);

    // 2. Draw the point
    ellipse(x, y, 4); // Draws a small red dot at the city's location
  }
}

function draw() {
  // No continuous animation needed for static data points
}
```

# Mapping Geographic Data to a 2D Canvas

This guide demonstrates how to transform raw geographic data (latitude and longitude) into visual points on a digital canvas using JavaScript and the p5.js library.

## The Logic of Coordinate Mapping [13:30]

To visualize data points, we must translate geographic coordinates into screen-space pixel coordinates ($x$ and $y$). This requires the `map()` function, which performs linear interpolation to scale values from one range to another.

### Mapping Longitude (X-axis)
Longitude represents the east-west position. In a standard coordinate system, this maps to the $x$ axis of our canvas.

```javascript
// Mapping longitude from a geographic range to the canvas width
let x = map(city.longitude, minLong, maxLong, 0, width);
```

### Mapping Latitude (Y-axis)
Latitude represents the north-south position. However, there is a critical distinction between geographic and screen coordinates:
*   **Geographic Latitude:** Increases as you move **North**.
*   **Screen Y-coordinates:** Increase as you move **Down** the screen.

To account for this inversion, when mapping latitude to the $y$ axis, you must swap the order of your minimum and maximum values so that the highest latitude (North) maps to a lower $y$ value (top of screen).

```javascript
// Mapping latitude from geographic range to canvas height
// Note: maxLat is passed before minLat to invert the Y-axis
let y = map(city.latitude, maxLat, minLat, 0, height);
```

## Iterative Boundary Search [13:30]

Before we can map coordinates, we need to know the boundaries of our dataset (the "bounding box"). We do this by iterating through every object in our data array and tracking the extreme values.

The following logic finds the minimum and maximum longitude and latitude across all cities:

```javascript
for (let city of cities) {
  if (city.longitude < minLong) {
    minLong = city.longitude;
  }
  if (city.latitude > maxLat) {
    maxLat = city.latitude;
  }
  if (city.longitude > maxLong) {
    maxLong = city.longitude;
  }
  if (city.latitude < minLat) {
    minLat = city.latitude;
  }
}
```

## Visualizing the Data [15:03]

Once the boundaries are established and coordinates are mapped, we can render the data. In this demonstration, we use `ellipse()` to draw small red dots representing each city.

```javascript
function draw() {
  background(0); // Black background

  for (let city of cities) {
    // Calculate x and y using the calculated boundaries
    let x = map(city.longitude, minLong, maxLong, 0, width);
    let y = map(city.latitude, maxLat, minLat, 0, height);
    
    fill(255, 0, 0); // Red color
    ellipse(x, y, 3); // Draw a dot with 3px diameter
  }
}
```

When executed, the canvas displays a cluster of red dots that form a recognizable shape—in this case, the outline of the United States.

## Handling Data Outliers and Proportions [15:40]

### Removing "Noise" from the Visualization
Sometimes, geographic datasets include locations that are far removed from the primary area of interest (e.g., Hawaii or Alaska when mapping the contiguous US). These outliers can compress your main data into a small, unreadable cluster.

To fix this, you can manually edit the source JSON file to remove these outliers or adjust your mapping boundaries to a specific "crop" of the data.

**Example: Editing `cities.json` in a text editor (like VS Code):**
If you remove the objects for Anchorage and Honolulu from your JSON array, the remaining points will occupy more of the visual space.

### The Aspect Ratio Problem [19:10]
A common issue in data visualization is the "squashed" effect. Because geographic coordinates (degrees) and digital pixels do not share the same aspect ratio, a square canvas will make a non-square country look distorted.

If you are mapping the US, which is wider than it is tall, a square canvas will make the map look vertically stretched. To solve this, you must define your canvas dimensions to match the proportions of your data's bounding box:

```javascript
// Instead of a square canvas, use dimensions that respect the data's shape
function setup() {
  createCanvas(600, 400); 
}
```

# Visualizing Geographic Data with p5.js

This resource explores the process of transforming raw geographic data into a visual representation on a digital canvas. We focus on the transition from abstract JSON objects to a "dotty" map of the United States, addressing the mathematical challenges of coordinate mapping and data boundaries.

## From Data to Visual Emergence [19:30]

When mapping geographic data, the goal is to translate coordinates into a visual structure that reflects real-world patterns. By iterating through a dataset of cities, we can begin to see the "shape" of a country emerge from raw numbers.

The following code demonstrates how we load a JSON file containing city data and iterate through it to find the extreme boundaries (minimum and maximum latitude) of our dataset.

```javascript
function preload() {
  // Load the JSON data asynchronously
  loadJSON("assets/cities.json", onLoad, onError);
}

function setup() {
  createCanvas(600, 400);
  console.log(cities); // Verify data is loaded in the console
}

// Iterating through cities to find spatial boundaries
for (let city of cities) {
  if (city.latitude < minLat) {
    minLat = city.latitude;
  }
  if (city.latitude > maxLat) {
    maxLat = city.latitude;
  }
}
```

As the code executes, the preview area displays a black background populated with small, scattered red dots. While these dots may initially look random, they begin to reveal the underlying structure of the United States: a high density of points on the East Coast, sparse areas in the middle, and clusters along the West Coast (California, Oregon, and the Pacific Northwest).

## The Challenge of Representation [20:00]

While the data points reveal a recognizable shape, this "dotty" version of the U.S. is not a perfect representation. There are several design and mathematical hurdles to overcome:

1.  **Missing Context:** The current visualization lacks an outline or borders, making it difficult to distinguish the shape of the country from the surrounding space.
2.  **The Connection Problem:** While one might consider drawing lines between cities to create an outline, this can be misleading. For example, connecting points in Florida to Southern Texas would create lines that cut across the map, misrepresenting the organic nature of geographic borders.
3.  **Proportionality:** The visual representation is currently limited by the arbitrary bounding box of our data, which may not match the aspect ratio of our digital canvas.

## Implementing Buffers for Spatial Padding [20:30]

A common issue in data visualization is "clipping," where data points fall exactly on the edge of the canvas and become difficult to see or are cut off. To solve this, we can implement a **buffer**—adding a set amount of "padding" to our coordinate boundaries.

Instead of mapping directly to the exact minimum and maximum values, we add or subtract a fixed degree value (e.g., 10 degrees) to ensure the data sits comfortably within the frame.

```javascript
function draw() {
  background(0);

  for (let city of cities) {
    // Mapping longitude with a buffer to prevent edge clipping
    let x = map(city.longitude, -123.2620435 + 10, -70.2553259 - 10, 0, width);
    // Mapping latitude with a buffer
    let y = map(city.latitude, 48.74908 - 10, 25.4687224 + 10, 0, height);
    
    ellipse(x, y, 3);
  }
}
```

### Refined Mapping with Precision Buffers [21:30]

The previous approach of adding a flat 10-degree buffer is a good start, but it may still result in distortion because the buffer isn't proportionate to the screen. A more refined approach involves using a smaller, more controlled buffer (e.g., 5 degrees) to tighten the representation while still providing enough breathing room for the data points.

```javascript
function draw() {
  background(0);

  for (let city of cities) {
    // Using a smaller buffer for more precise spatial representation
    let x = map(city.longitude, -123.2620435 - 5, -70.2553259 + 5, 0, width);
    let y = map(city.latitude, 48.74908 + 5, 25.4687224 - 5, 0, height);
    
    ellipse(x, y, 3);
  }
}
```

By applying these transformations, we move from raw numbers to a meaningful spatial visualization. The next step in this progression is to import secondary datasets—such as population or density information—to add qualitative depth to our quantitative map.
