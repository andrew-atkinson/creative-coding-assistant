# Audio Analysis and Frequency Domains with FFT

This resource explores how to use the Fast Fourier Transform (FFT) in p5.js to decompose audio into its constituent frequencies, enabling the creation of visual representations like spectrograms and frequency-reactive animations.

## Understanding the FFT [00:36]

To visualize sound, we must move from the **time domain** (amplitude over time) to the **frequency domain** (amplitude across different pitches). In p5.js, this is handled by the `p5.FFT` class.

### The Frequency Domain
The "frequency domain" refers to the different pitches present in a sound. When we analyze audio using FFT, we are essentially "chopping up" the sound into discrete buckets, or **bins**, ranging from the lowest possible pitch to the highest.

In a standard setup:
```javascript
fft = new p5.FFT(0.8, 512);
```
The second parameter (`512`) defines the number of bins. This means the audio spectrum is divided into 512 equal segments.

### Human Hearing and Frequency Ranges [02:30]
The standard human hearing range is approximately **20 Hz to 20,000 Hz**. When we use an FFT with 512 bins, the algorithm distributes those 512 buckets evenly across that entire range.

Understanding these ranges is crucial for creative coding, as it allows you to target specific "musical" or "vocal" zones:

| Frequency Range | Description / Musical Context |
| :--- | :--- |
| **32 Hz – 512 Hz** | Rhythm frequencies, lower bass notes, and sub-bass. |
| **512 Hz – 2048 Hz** | Speech intelligibility; the core of the vocal range. |
| **2048 Hz+** | High-frequency textures and technical speech components. |

For musical context, consider the range of a piano: it spans from roughly **32 Hz** (the lowest C) to about **4,000 Hz**.

## Implementing a Spectrogram [00:00]

A spectrogram is a visual representation of the spectrum of frequencies in a sound as it varies with time. We can create one by iterating through the FFT bins and drawing rectangles for each frequency level.

### Basic Implementation
The following code demonstrates how to capture audio from a file and map the amplitude of each frequency bin to a vertical position on the canvas.

```javascript
let sound;
let fft;
let bins;
let x = 0;

function preload() {
  // Load the audio file to be analyzed
  sound = loadSound("assets/pelagibacter.mp3");
}

function setup() {
  createCanvas(400, 512);
  // Initialize FFT with a smoothing factor of 0.8 and 512 bins
  fft = new p5.FFT(0.8, 512);
  fft.setInput(sound);
  sound.play();
  noStroke();
}

function draw() {
  // bins is an array of 512 amplitude values (0 to 1)
  bins = fft.analyze();

  for (let y = 0; y < bins.length; y++) {
    // Map the amplitude of the bin to a color/fill
    fill(bins[y]); 
    // Draw a 1x1 pixel rectangle at the current x position for each frequency bin
    rect(x, y, 1, 1);
  }

  // Move the drawing cursor to the right for the next frame
  x++;

  // Reset x when it reaches the edge of the canvas to create a scrolling effect
  if (x > width) {
    x = 0;
  }
}
```

### Visualizing the Data
When this code runs, the `bins` array contains 512 values. As the loop iterates through `y`, it draws a vertical column of pixels. Because we increment `x` every frame, the spectrogram "scrolls" across the screen, creating a history of the audio frequencies.

## Targeting Specific Frequencies [03:19]

Once you can visualize the entire spectrum, you can use specific indices within the `bins` array to drive individual visual elements. 

Because the bins are distributed across the frequency range, different indices correspond to different pitches:
*   **Low Indices:** Represent bass and low-frequency sounds.
*   **High Indices:** Represent treble and high-pitched sounds.

By isolating a single index—for example, `bins[13]` or `bins[221]`—you can map the amplitude of that specific frequency to a property like the diameter of a circle. This creates an animation that responds only to a specific pitch, rather than the entire soundscape.

## Advanced Analysis: Octave Bands [05:21]

While raw FFT bins are mathematically accurate, they aren't always "musically" intuitive because human hearing is logarithmic (we perceive pitch in octaves). 

To create more organic or musically relevant visualizations, you can use higher-level abstractions provided by the `p5.sound` library:

*   **`fft.getOctaveBands(n)`**: Groups the frequency bins into $n$ octave bands, providing a more musically structured data set.
*   **`fft.logAverages()`**: Provides smoothed, logarithmic averages of the frequency data, which is much more useful for driving smooth visual movements than the "jittery" raw FFT bins.

# Audio Analysis and Visualization with FFT

This guide explores how to use the Fast Fourier Transform (FFT) in `p5.js` to transform audio signals into visual data. We will cover the distinction between frequency-domain and time-domain analysis and demonstrate how to map these data points to visual elements.

## Understanding the Spectrogram [07:30]

A spectrogram is a visual representation of the spectrum of frequencies in a sound as they vary with time. In `p5.js`, we can generate this by iterating through the frequency "bins" provided by the FFT and drawing them as vertical lines or rectangles.

### Implementing a Basic Spectrogram
To create a spectrogram, we use `fft.analyze()` to get an array of amplitude values for different frequency bins. We then loop through this array and draw a rectangle at the current `x` position for each bin.

```javascript
let sound;
let fft;
let bins;
let x = 0;

function preload() {
  sound = loadSound("assets/pelagibacter.mp3");
}

function setup() {
  createCanvas(400, 512);
  // The second parameter defines the number of frequency bins (e.g., 512)
  fft = new p5.FFT(0.8, bins); 
  fft.setInput(sound);
  sound.play();
  noStroke();
}

function draw() {
  // Using a low alpha in background allows for a "trail" effect
  background(0, 5); 
  
  bins = fft.analyze();

  for (let y = 0; y < bins.length; y++) {
    fill(bins[y]); // The amplitude value determines the brightness/color
    rect(x, y, 1, 1);
  }

  x++;
  if (x > width) {
    x = 0;
  }
}
```

The canvas shows a vertical pattern of varying intensities. As `x` increments, the spectrogram "scrolls" or builds up across the screen.

## Mapping Specific Frequencies [08:30]

Once you have access to the `bins` array, you can isolate specific frequencies to drive individual visual elements. For example, instead of drawing the whole spectrum, you can use a single bin to control the size of a circle.

### Isolating Frequency Bins
By using `console.log(bins)` or checking the amplitude at a specific index, you can find which bin corresponds to a certain pitch. If a note is detected at the 13th bin, you can map `bins[13]` to a visual property.

```javascript
function draw() {
  background(0, 5);
  bins = fft.analyze();

  // Draw the spectrogram
  for (let y = 0; y < bins.length; y++) {
    fill(bins[y]);
    rect(x, y, 1, 1);
  }

  // Isolate specific frequencies using push() and pop()
  // This prevents the stroke/fill settings from affecting the spectrogram loop
  push();
  stroke(255, 0, 0); // Red stroke
  noFill();
  // Use a specific bin index (e.g., 13) to drive the diameter
  circle(width/2, 40, bins[13]); 
  pop();

  x++;
  if (x > width) { x = 0; }
}
```

### Managing Visual Layers with `push()` and `pop()`
When drawing multiple elements, one element's style (like `stroke` or `noFill`) can "bleed" into the next drawing command. To prevent this, use:
*   `push()`: Saves the current drawing state (color, stroke, transformations).
*   `pop()`: Restores the last saved state.

This is crucial when you want to draw a circle with a red stroke without turning the entire spectrogram into red lines.

### Adding Variance
To create more complex visuals, you can map multiple frequencies to different elements at different positions:

```javascript
push();
stroke(255, 0, 0); // Red for low frequency
noFill();
circle(width/2, 40, bins[13]);

stroke(255, 128, 0); // Orange for higher frequency
circle(width/2, 300, bins[221]);
pop();
```

## Time-Domain Analysis: Waveforms [11:06]

While `fft.analyze()` provides data in the **frequency domain** (pitch), `fft.waveform()` provides data in the **time domain** (amplitude over time).

### The Waveform Concept
The waveform represents the actual oscillation of the sound wave. It returns an array where each value represents the amplitude at a specific moment in time. 

**Key Technical Detail:** The values in a waveform array typically range from **-1 to 1**.
*   **0** represents the center (silence/equilibrium).
*   **Positive values** represent the peak of a wave.
*   **Negative values** represent the trough of a wave.

The "loudness" is determined by how far the value deviates from zero, rather than whether it is positive or negative.

### Visualizing the Waveform
To visualize this, you can map the waveform values to a line or a series of points.

```javascript
let waveform;

function draw() {
  background(0, 5);
  bins = fft.analyze();
  // ... (spectrogram code)

  waveform = fft.waveform();

  push();
  stroke(255, 0, 0);
  noFill();
  // Draw a circle driven by the waveform amplitude at a specific point
  circle(width/2, 40, bins[13]); 
  pop();

  // To draw a full waveform line, you would loop through the 'waveform' array
  // and use line() or vertex() to connect the points.
}
```

By combining the frequency-based "brightness" of a spectrogram with the time-based oscillation of a waveform, you can create highly responsive and organic generative art.

# Audio Analysis and Visualization with FFT

This guide explores how to use the Fast Fourier Transform (FFT) in `p5.js` to bridge the gap between raw audio data and expressive generative art. We will move from basic frequency analysis to more sophisticated, perceptually-driven methods like spectral centroids and octave bands.

## Visualizing the Spectrogram [14:28]

The first step in audio visualization is decomposing sound into its constituent frequencies. Using `fft.analyze()`, we can retrieve an array of amplitude values (bins) representing different pitches.

To create a **spectrogram**—a visual history of frequency intensity—we iterate through these bins and draw them as vertical rectangles.

```javascript
// Inside the draw() function
bins = fft.analyze();

for (let y = 0; y < bins.length; y++) {
  // We use a low alpha (e.g., 50) to create a "trail" effect
  // This prevents the new frames from completely wiping out previous ones
  fill(bins[y], 50); 
  rect(x, y, 1, 1);
}
```

The canvas shows a vertical column of colored pixels on the left side, where the color intensity corresponds to the volume of specific frequencies.

## Time-Domain vs. Frequency-Domain [15:00]

It is crucial to distinguish between the two primary ways we analyze audio data:

1.  **Frequency Domain (`fft.analyze()`):** Decomposes sound into discrete frequency "bins" (low to high pitches). This is ideal for creating spectrograms or driving animations based on specific notes.
2.  **Time Domain (`fft.waveform()`):** Analyzes amplitude over a brief moment in time, representing the actual oscillation of the sound wave.

### Implementing a Waveform "Equalizer" [15:08]

To visualize the waveform, we can draw a vertical line that responds to the amplitude. Since `waveform()` returns values between -1 and 1, we must scale these values by the canvas height to make them visible.

```javascript
waveform = fft.waveform();

push();
  stroke(255, 128, 0, 128); // Orange with transparency
  translate(0, height/2);   // Center the line vertically
  // We draw from negative waveform to positive waveform 
  // so the line expands outward from the center
  line(x, -waveform[0] * height, x, waveform[0] * height);
pop();
```

**Technical Insight: Managing Drawing Order and Transparency**
When drawing multiple elements (like the waveform line and concentric circles), you may find that one element "overwrites" another. To prevent this, use `push()` and `pop()` to isolate transformations and styles. Additionally, using a lower alpha value (transparency) in your `fill()` or `stroke()` allows different visual layers to blend together, creating a more organic, glowing effect rather than harsh, overlapping shapes.

## Mapping Specific Frequencies [16:30]

Instead of visualizing the whole spectrum, you can drive specific shapes using individual frequency bins. This creates an animation that responds only when a certain pitch is hit.

```javascript
push();
  stroke(255, 0, 0); // Red
  noFill();
  // bins[13] targets a specific low-mid frequency
  circle(width/2, 40, bins[13]); 
  
  stroke(255, 128, 0); // Orange
  // bins[221] targets a specific high frequency
  circle(width/2, 300, bins[221]); 
pop();
```

The canvas shows two concentric circles that pulse or change size based on the audio input, specifically reacting to the frequencies mapped to indices 13 and 221.

## Moving Toward Human Perception [17:00]

While raw frequency bins are mathematically accurate, they don't always align with how humans perceive sound. To create more "musical" or "expressive" art, we can use higher-level abstractions.

### The Spectral Centroid [17:25]
The **spectral centroid** indicates the "center of mass" of the spectrum. Perceptually, this correlates to the **brightness** or **timbre** of a sound. A high spectral centroid sounds "bright" or "sharp," while a low one sounds "dark" or "muffled." Using this value allows you to map audio characteristics directly to aesthetic qualities like color temperature or light intensity.

### Octave Bands [18:15]
Human hearing is logarithmic, not linear. The `getOctaveBands()` method provides a way to group frequencies into musically relevant ranges.

In `p5.js`, this function defaults to **one-third octave bands** (using a parameter $N=3$). If you pass `1` as the parameter, it will return data grouped by whole octaves.

```javascript
// Accessing octave bands from the FFT object
let bands = fft.getOctaveBands(1); 
```

By using these grouped averages rather than raw, jittery frequency bins, you can achieve much smoother and more musically intuitive visual movements.

# Audio Analysis: Octave Bands and Logarithmic Averages

This guide explores how to move beyond raw frequency data in p5.js by using octave bands and logarithmic averaging. This transition allows for more musically intuitive visualizations, moving from "jittery" individual frequency bins to smoother, broader strokes that represent musical pitch ranges.

## Understanding Octave Bands [21:00]

When using the `p5.FFT` class, a standard analysis returns an array of many individual frequency bins. While precise, this data can be too granular for expressive art. To group these frequencies into musically relevant sections, we use **octave bands**.

By calling `fft.getOctaveBands(1)`, we decompose the audio into 11 distinct octaves. Each object in the resulting array represents a specific frequency range (e.g., bass, mid-range, treble).

```javascript
// Get octave bands (1 represents the number of octaves to return)
let octaveBands = fft.getOctaveBands(1);
```

When inspecting this data via `console.log()`, you will see an array of objects. Each object contains the frequency data for that specific octave, providing a much more structured way to interact with sound than raw bins.

## Smoothing Data with Logarithmic Averages [22:00]

While octave bands group frequencies, the raw data within those bands can still be erratic. To create organic visual movement, we need to calculate the average amplitude across those bands.

The `fft.logAverages()` method is designed for this purpose. It takes the octave bands as an input and returns an array of average amplitude values across the spectrum.

### Implementation Workflow
To implement this, you first capture the octave bands and then pass them into the logarithmic average function:

```javascript
// 1. Capture the octave bands
let octaveBands = fft.getOctaveBands(1);

// 2. Calculate the logarithmic averages of those bands
let octaveAverages = fft.logAverages(octaveBands);
```

The resulting `octaveAverages` array is significantly more useful for generative art. Instead of reacting to a single, high-frequency spike, your visuals will react to the "energy" within a musical range.

## Mapping Data to Visual Elements [24:30]

Once you have the `octaveAverages` array, you can map its values to visual properties like circle diameter or color.

### Managing Visual State with `push()` and `pop()`
When drawing multiple elements—especially when using transformations like `translate()` or changing styles with `fill()` and `stroke()`—it is critical to use `push()` and `pop()`. This ensures that the stylistic changes applied to one element (like a specific color or position) do not "bleed" into the next drawing command.

### Creating Iterative Visuals
To create a series of shapes that respond to the different musical ranges, you can loop through the `octaveAverages` array. By using the index of the loop, you can map the "pitch" (low to high) to a vertical position on the screen.

```javascript
push();
for (let i = 0; i < octaveAverages.length; i++) {
  // Map the index (0-10) to a vertical position on the canvas
  let yPos = map(i, 0, octaveAverages.length, 0, height);
  
  // Use the amplitude (octaveAverages[i]) to drive size or color
  let amplitude = octaveAverages[i];
  
  fill(amplitude, 100); // Example: amplitude drives brightness
  circle(width / 2, yPos, 50);
}
pop();
```

### Advanced Color Mapping [26:30]
You can create sophisticated color gradients by normalizing the index of the array. By dividing the current index `i` by the total length of the array, you create a value between 0 and 1. This "percentage" can then be used to transition through colors (e.g., from dark blue to bright red) as you move from low to high frequencies.

```javascript
push();
for (let i = 0; i < octaveAverages.length; i++) {
  // Create a color gradient based on the index (0 to 1 range)
  let colorRatio = i / octaveAverages.length;
  fill(colorRatio * 255, 0, 100); // Transitions from dark to bright red/pink
  
  // Map the amplitude to a vertical position
  let yPos = map(i, 0, octaveAverages.length, 0, height);
  circle(width / 2, yPos, octaveAverages[i]);
}
pop();
```

In this setup:
- **Low indices** (low frequencies/bass) appear at the top of the screen.
- **High indices** (high frequencies/treble) appear at the bottom of the screen.
- **The color** shifts dynamically based on where you are in the musical spectrum.

# Audio Analysis and Visualization with FFT

This resource explores how to use the Fast Fourier Transform (FFT) in `p5.js` to transform audio signals into dynamic visual elements. We move from raw frequency data to more sophisticated, musically intuitive abstractions like octave bands and spectral centroids.

## Mapping Specific Frequencies [27:47]

A common technique in generative art is to isolate specific frequency "bins" from the FFT analysis and map them to visual properties like diameter or position. This allows a specific pitch (e.g., a bass note or a high whistle) to drive a specific visual element.

To do this, you access the `bins` array returned by `fft.analyze()`. By selecting a specific index, you can create an animation that responds only to that frequency range.

```javascript
// Animation at particular frequencies
push();
  stroke(255, 0, 0); // Red stroke
  noFill();
  // Map the amplitude of bin index 13 to circle diameter
  circle(width/2, 40, bins[13]); 

  stroke(255, 128, 0); // Orange stroke
  // Map the amplitude of bin index 221 to circle diameter
  circle(width/2, 300, bins[221]); 
pop();
```

The canvas shows two circles at different vertical positions. As the audio plays, one circle expands and contracts based on low-frequency energy (index 13), while the other reacts to higher-frequency energy (index 221).

## Using Octave Bands for Organic Movement [28:05]

While raw frequency bins are useful, they can be erratic and "jittery" because they represent very narrow slices of sound. For more organic, musical movement, it is better to use **octave bands**.

By using `fft.getOctaveBands()` and `fft.logAverages()`, we can group frequencies into musically relevant groups (octaves) and smooth the data. This provides a "graphic equalizer" effect that feels more natural to the human ear.

```javascript
// Get octave bands and smooth them using logarithmic averages
octaveBands = fft.getOctaveBands(1);
octaveAverages = fft.logAverages(octaveBands);

push();
for (let i in octaveAverages) {
  // Map the index to a vertical position and amplitude to diameter
  fill(50, 0, (i / octaveAverages.length) * 255);
  circle(width/2, (i / octaveAverages.length) * height, octaveAverages[i]);
}
pop();
```

### Managing Visual Layers and Transparency [29:01]

When layering multiple visualizations—such as a spectrogram, an equalizer, and geometric shapes—the drawing order becomes critical. If you draw the background or a large shape last, it will "blow out" (overwrite) everything drawn before it.

To create depth and allow elements to blend, use low alpha (transparency) values in your `fill()` or `stroke()` commands.

```javascript
// Example of a translucent graphic equalizer
waveform = fft.waveform();
push();
  stroke(255, 128, 0, 32); // Very low alpha (32) for translucency
  translate(0, height/2);
  line(x, -waveform[0] * height, x, waveform[0] * height);
pop();
```

By using a low alpha value like `32` or even `5`, you allow the underlying frequency data to "bleed through," creating a more complex, layered aesthetic.

## Advanced Perceptual Metrics [30:14]

Beyond simple amplitude, the `p5.sound` library provides metrics that correlate to how humans actually perceive sound:

*   **Spectral Centroid:** This represents the "center of mass" of the spectrum. In musical terms, it is often associated with the **brightness** or timbre of a sound.
*   **Energy:** A measure of the overall power within specific frequency ranges.

You can use these values to drive non-linear animations. For example, instead of a circle simply getting larger with volume, you could use the **spectral centroid** to move the circle up and down on the Y-axis. As the sound becomes "brighter" (higher centroid), the visual element moves higher on the screen.

```javascript
// Conceptual implementation of using centroid for movement
let brightness = fft.getCentroid(); 
let yPos = map(brightness, 0, sampleRate/2, height, 0);
circle(width/2, yPos, amplitude);
```

## Summary of FFT Methods

| Method | Purpose | Visual Application |
| :--- | :--- | :--- |
| `fft.analyze()` | Returns raw frequency bins | Spectrograms, specific pitch triggers |
| `fft.waveform()` | Returns time-domain amplitude | Oscilloscope/Waveform lines |
| `fft.getOctaveBands()` | Groups frequencies into octaves | Musical "equalizer" style visuals |
| `fft.logAverages()` | Smooths frequency data | Organic, undulating shapes |
| `fft.getCentroid()` | Measures spectral "brightness" | Driving movement or color shifts |
