# Visualizing Geographic Data: JSON vs. Tabular Formats

This lesson explores how different data structures—specifically hierarchical objects and tabular values—can be used to drive generative visualizations. We begin by analyzing a point-based map of the United States and then contrast the structure of JSON with that of CSV/TSV formats.

## Visualizing Points with JSON [00:00]

The current visualization uses a JSON dataset of cities to create an "impressionistic" map. By plotting individual city coordinates, the canvas renders a scattered pattern of red dots that roughly outline the contiguous 48 US states.

### Data Normalization and Bounds
To map geographic coordinates (latitude and longitude) onto a digital canvas, we must first determine the boundaries of our dataset. We do this by iterating through the data to find the minimum and maximum values for both axes. This "bounding box" is essential for translating real-world coordinates into local pixel values.

The following code demonstrates how to load a JSON file and calculate these bounds:

```javascript
// Cities JSON 
// https://gist.github.com/Miserlou/c5cd8364bf9b2420bb29
let cities;

// Initialize bounds with extreme values to ensure any data point 
// will correctly expand the range during iteration.
let minLat = 90;
let maxLat = -90;
let minLong = 180;
let maxLong = -180;

function preload() {
  // Asynchronously load the JSON data before setup runs
  loadJSON("assets/cities.json", onLoad, onError);
}

function setup() {
  createCanvas(500, 400);
  console.log({cities});

  // Iterate through the array of city objects to find data extremes
  for (let city of cities) {
    if (city.latitude < minLat) {
      minLat = city.latitude;
    }
    if (city.latitude > maxLat) {
      maxLat = city.latitude;
    }
    if (city.longitude < minLong) {
      minLong = city.longitude;
    }
    if (city.longitude > maxLong) {
      maxLong = city.longitude;
    }
  }
}
```

## Understanding Data Structures [00:27]

To expand our generative art beyond simple dots, we need to understand the different ways data can be organized.

### JSON (JavaScript Object Notation)
JSON is a format used to represent hierarchical data through objects. In our example, each city is an object containing multiple key-value pairs (such as `latitude` and `longitude`). This structure is highly flexible, allowing each entry to contain complex, nested information.

### CSV and TSV (Tabular Data)
In contrast to the hierarchical nature of JSON, **CSV** (Comma Separated Values) and **TSV** (Tab Separated Values) represent **tabular data**. 

Think of these formats as a digital version of a Google Sheets or Excel spreadsheet. The data is organized into:
* **Rows:** Each row represents a single record or entry.
* **Columns:** Each column represents a specific attribute (often defined by a header row at the top).

While JSON is excellent for complex, nested relationships, tabular formats like CSV/TSV are often more efficient for large-scale datasets—such as census or demographic information—where you want to map many different variables across a massive number of entries.
