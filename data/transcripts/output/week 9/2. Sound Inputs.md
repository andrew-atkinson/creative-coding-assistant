# Building a Real-Time Spectrogram with p5.js

A spectrogram is a visual representation of the various frequencies present in a sound signal over time. By using the **Fast Fourier Transform (FFT)**, we can decompose an audio signal into a series of discrete frequency "bins," allowing us to map mathematical data to visual properties like height, color, or position.

## Asset Management and Asynchronous Loading [01:31]

Before we can analyze sound, we must load it into our sketch. When working with external files in p5.js, it is crucial to understand the difference between `setup()` and `preload()`.

### The Importance of `preload()`
The `setup()` function in p5.js runs asynchronously, meaning the program will continue to execute the `draw()` loop even if the audio file hasn't finished downloading. If you attempt to play a sound before it is fully loaded, the sketch will break.

To solve this, use the `preload()` function. Anything defined inside `preload()` is handled synchronously; p5.js will ensure the asset is fully loaded before moving on to `setup()`.

```javascript
let sound; // Global variable to hold the audio object

function preload() {
  // Load the sound file from the assets folder
  sound = loadSound("assets/pelagibacter.mp3");
}

function setup() {
  createCanvas(400, 400);
}

function draw() {
  background(220);
}
```

**Note on File Paths:** Ensure your file path is exact. If your sound file is inside a folder named `assets`, the string must be `"assets/filename.mp3"`.

## Implementing Fast Fourier Transform (FFT) [04:15]

Once the sound is loaded, we can use the `p5.FFT` class to analyze it. The FFT takes a time-domain signal (the raw audio wave) and converts it into the frequency domain.

### Initializing the FFT Object
To use the FFT, we create a new instance of `p5.FFT`. This object requires an input (the sound) to analyze.

```javascript
let sound;
let fft;

function preload() {
  sound = loadSound("assets/pelagibacter.mp3");
}

function setup() {
  createCanvas(400, 400);
  
  // Initialize FFT with smoothing and bin count
  // fftSize must be a power of two between 16 and 1024
  fft = new p5.FFT(0.8, 512); 
  
  // Connect the sound object to the FFT analyzer
  fft.setInput(sound);
  
  sound.play();
}

function draw() {
  background(220);
}
```

### Understanding FFT Parameters and Methods
The `p5.FFT` constructor typically takes parameters for smoothing and the number of bins (the "size" of the FFT). A higher bin count provides more granular frequency data but requires more processing power.

The `p5.sound` documentation reveals several powerful methods for analyzing audio:
*   **`analyze()`**: Returns an array of values representing the magnitude of each frequency bin.
*   **`getCentroid()`**: Returns the "center" of the sound—the dominant frequency.
*   **`getEnergy()`**: Provides data on the energy levels across different octave bands.

## Extracting Frequency Data [07:32]

To visualize the sound, we need to extract the data from the FFT object. We do this by calling `fft.analyze()` inside the `draw()` loop, which runs every frame. This provides us with an array of "bins," where each bin represents the amplitude (volume) of a specific frequency range.

```javascript
let sound;
let fft;

function preload() {
  sound = loadSound("assets/pelagibacter.mp3");
}

function setup() {
  createCanvas(400, 400);
  fft = new p5.FFT(512);
  fft.setInput(sound);
  sound.play();
}

function draw() {
  background(220);
  
  // Capture the current frequency data into an array called 'bins'
  let bins = fft.analyze();
  
  // The 'bins' array now contains the amplitude for each frequency segment.
  // We can iterate through this array to draw our spectrogram.
}
```

By capturing these `bins` every frame, we can map the amplitude of each frequency to a visual element, creating a dynamic, real-time representation of the audio.

# Visualizing Sound with Fast Fourier Transform (FFT)

This guide demonstrates how to use the **p5.js** library and its `p5.sound` module to perform real-time audio analysis. We will transform an audio signal from the time domain into the frequency domain to create a scrolling spectrogram.

## Understanding the FFT [08:00]

Before coding, it is essential to understand what a **Fast Fourier Transform (FFT)** actually does. In digital signal processing, sound is typically represented as a waveform over time (the time domain). The FFT is an algorithm that decomposes this complex waveform into its constituent frequencies (the frequency domain).

As shown in the technical diagrams, an FFT takes a single oscillating signal and breaks it down into discrete vertical bars representing different frequencies. In our implementation, we define a specific number of "bins"—discrete segments of the frequency spectrum—to analyze. For this project, we will use **512 bins**.

## Implementing Audio Analysis [08:30]

To analyze sound, we use the `p5.FFT` class. When initializing the FFT, it is highly recommended to include a **smoothing** parameter. Without smoothing, the data can be too erratic for visual representation; adding a value like `0.8` helps create more fluid transitions between frames.

### The Basic Setup
To ensure audio assets are ready before the program attempts to play them, we use the `preload()` function.

```javascript
let sound;
let fft;
let bins;

function preload() {
  // Load the audio file into a variable
  sound = loadSound("assets/pelagibacter.mp3");
}

function setup() {
  createCanvas(400, 400);
  // Initialize FFT with smoothing (0.8) and bin count (512)
  fft = new p5.FFT(0.8, 512);
  // Tell the FFT which sound source to analyze
  fft.setInput(sound);
  sound.play();
}

function draw() {
  background(220);
  // Analyze the sound and store results in the 'bins' array
  bins = fft.analyze();
}
```

### Debugging with Console Logs
When working with complex mathematical data like FFT bins, it is helpful to use `console.log(bins)` to inspect the raw data. Each element in the resulting array is a number representing the amplitude of a specific frequency bin.

If you want to inspect the data without the program running indefinitely, you can use `noLoop()` combined with a frame counter to freeze the execution after a few frames:

```javascript
function draw() {
  background(220);
  bins = fft.analyze();
  console.log(bins);

  // Stop the loop after 10 frames to inspect the console
  if (frameCount === 10) {
    noLoop();
  }
}
```

## Mapping Frequency to Visuals [10:25]

Once we have the `bins` array, we can map those values to visual properties. To create a spectrogram, we iterate through the bins and draw shapes based on their amplitude.

### The Logic of the Spectrogram
1.  **Frequency Mapping**: We map the index of the array to the Y-axis. In this implementation, we treat the top of the screen (Y = 0) as the lowest frequency and move downward as frequencies increase.
2.  **Amplitude Mapping**: We use the value of each bin to determine the visual "intensity" or length of a shape.
3.  **The Stroke Issue**: When drawing very thin lines (e.g., 1 pixel wide), the default `stroke()` can overlap and obscure the data. Using `noStroke()` is essential for a clean visualization.

### Building the Visualizer
We use a `for` loop to iterate through every bin in the array. We will draw a rectangle for each frequency, where the width of the rectangle is driven by the amplitude.

```javascript
function draw() {
  background(220);
  bins = fft.analyze();

  // Iterate through the bins array
  for (let y = 0; y < bins.length; y++) {
    // Use the bin value to set the color/brightness
    fill(bins[y]);
    // Draw a rectangle: x, y, width, height
    // We use 'width' for the rect to create a horizontal bar effect
    rect(0, y, width, 1);
  }
}
```

### Refining the Visual Output [14:30]
To make the visualization more impactful, we can manipulate how the data is scaled. For example, if the audio signal only occupies a small portion of the frequency spectrum (common with orchestral strings), we can scale the Y-coordinate to "stretch" the visualization across the canvas.

```javascript
function draw() {
  background(220);
  bins = fft.analyze();

  for (let y = 0; y < bins.length; y++) {
    fill(bins[y]);
    // Scale the Y position and height to fill more of the canvas
    rect(0, y * 2, width, 1); 
  }
}
```

When this code runs, the preview pane displays a series of horizontal bars. The brightness and length of these bars change in real-time as the frequencies in the audio file fluctuate, creating a dynamic spectral profile of the sound.

# Visualizing Sound: Creating a Real-Time Spectrogram in p5.js

This guide demonstrates how to use the Fast Fourier Transform (FFT) to convert audio signals into a visual spectrogram. We will move from analyzing static audio files to capturing real-time microphone input, creating a scrolling visual representation of frequency and amplitude.

## Building the Spectrogram Logic [15:00]

To create a spectrogram, we need to map audio data (frequencies) to visual properties. Instead of drawing the entire waveform at once, we will draw thin vertical lines that progress across the screen over time.

### Mapping Frequencies to Space
A standard spectrogram maps frequency to the vertical axis ($y$) and time to the horizontal axis ($x$). To achieve this, we iterate through the frequency "bins" provided by the FFT and draw a rectangle for each.

```javascript
function setup() {
  createCanvas(400, 400);
  // Initialize FFT: (smoothing, bin count)
  fft = new p5.FFT(0.8, 512);
  fft.setInput(sound);
  sound.play();
  noStroke(); // Essential: prevents 1px strokes from obscuring data
}

function draw() {
  // background(220); // Commented out to allow the drawing to persist/accumulate
  bins = fft.analyze();

  for (let y = 0; y < bins.length / 2; y++) {
    fill(bins[y]); // Map amplitude to brightness/color
    rect(x, y * 2, 1, 2); // Draw a thin line at the current x position
  }

  // Handle temporal progression (scrolling)
  x++; 
  if (x > width) {
    x = 0; // Reset to the left side when reaching the edge
  }
}
```

### Key Implementation Details
*   **The `noStroke()` Command**: When drawing 1-pixel wide rectangles to represent frequency data, the default stroke can overlap and visually overwhelm the actual data. Using `noStroke()` ensures only the `fill` (the amplitude) is visible.
*   **The Scrolling Effect**: By using a global variable `x` that increments every frame and resets to `0` once it exceeds the canvas width, we create a continuous "wrap-around" scrolling effect.
*   **The `bins` Array**: The `fft.analyze()` function returns an array of values representing the magnitude of different frequency segments. We iterate through these to determine how "bright" or "tall" each visual element should be.

## Transitioning to Real-Time Microphone Input [17:00]

While audio files are useful for testing, real-time interaction is achieved by using the `p5.AudioIn` class to capture microphone input.

### Implementing Microphone Input
Unlike an MP3 file, a microphone does not need to be loaded via `preload()`. Instead, we initialize the microphone object in `setup()` and pass it to the FFT.

```javascript
let mic;
let fft;
let bins;
let x = 0;

function setup() {
  createCanvas(400, 400);
  
  // Create the microphone input object
  mic = new p5.AudioIn();
  mic.start(); // Crucial: must explicitly start the microphone
  
  // Initialize FFT and set its input to the mic
  fft = new p5.FFT(0.8, 512);
  fft.setInput(mic);
  
  noStroke();
}

function draw() {
  bins = fft.analyze();

  for (let y = 0; y < bins.length; y++) {
    fill(bins[y], 1, 2); // Using bins[y] to drive brightness
    rect(x, y, 1, 2);
  }

  x++;
  if (x > width) {
    x = 0;
  }
}
```

**Note on Browser Security:** When using the microphone, modern browsers require user interaction (like a click) to grant permission for audio capture.

## Artistic Context: The Spectral Profile [19:00]

The visual output of a spectrogram is heavily dependent on the "spectral profile" of the sound source. 

### Synthesized vs. Natural Sounds
*   **Synthesized Sounds:** Early electronic music pioneers like **Daphne Oram** (co-founder of the BBC Radiophonic Workshop) and **Delia Derbyshire** worked with early synthesizers that produced very clear, distinct frequency profiles. In a spectrogram, these appear as neat, rhythmic, and highly structured patterns of light.
*   **Natural/Complex Sounds:** Human voices or complex musical recordings produce a much denser, more chaotic array of frequencies.

### Historical Connection: The Oramics Machine
Daphne Oram invented the **Oramics machine**, a device that allowed her to draw images on film, which the machine then converted into sound. This is essentially a reverse spectrogram: instead of converting sound to an image, she used images to generate sound waves. Understanding the relationship between a visual pattern and its corresponding frequency is fundamental to both digital signal processing and generative art.

# Audio-Visual Synthesis: From Sound to Spectrograms

This lesson explores the relationship between sound and its visual representation, specifically focusing on how to use mathematical analysis to create generative art from audio signals.

## The Aesthetic of the Spectrogram [23:00]

A spectrogram is a visual representation of the spectrum of frequencies in a sound as they vary with time. When coding these visualizations, there is often an "aesthetic relationship" between the sound being played and the resulting visual output.

The instructor demonstrates this by playing various audio sources—ranging from synthesized sounds to natural recordings like a garden warbler—and observing how the frequency patterns manifest on the canvas.

The visual output typically appears as a high-contrast, grayscale pattern where:
*   **Vertical patterns/streaks** represent the intensity of specific frequencies.
*   **Horizontal bands or noise-like textures** represent the distribution of energy across the spectrum over time.

Finding an interesting visualization is a key part of the creative process; the goal is to ensure the visual function is as compelling and complex as the audio input driving it.

## The Evolution of Sound Manipulation [25:10]

To understand modern digital audio, it is helpful to look at the history of how sound has been manipulated.

### Traditional Tape Recording
Historically, changing the pitch of a recording was tied to the physical speed of the tape. To make a sound higher in pitch, you would run the tape twice as fast. However, this created a direct trade-off: increasing the pitch by an octave meant the recording would only last half as long.

### Sound Granulation [25:15]
In the 1980s, a technique called **sound granulation** emerged. This changed the fundamental relationship between frequency and pitch.

Instead of treating a sound wave as one continuous entity, granulation involves:
1.  Breaking the audio into tiny "grains" (e.g., 10-millisecond chunks).
2.  Reassembling these grains in new patterns.

This technique allows for the manipulation of pitch and time independently—a concept that paved the way for modern technologies like Auto-Tune. The instructor highlights the work of composers like Paul Lansky, who used these methods to create complex, synthetic compositions that move beyond simple "idle chatter."

## Moving Toward Integrated Media [26:34]

Contemporary composition often integrates sound with other forms of media. The instructor notes that modern composers (such as Adem Ilhan) create works where the audio is deeply integrated with visual or other sensory elements, moving away from simple "input-to-image" mapping toward a more holistic, integrated experience.

As you develop your own generative art, the challenge is to move beyond simply turning an input into a basic image. The goal is to create a visual system that responds meaningfully to the nuances of the sound, creating a cohesive aesthetic experience.
